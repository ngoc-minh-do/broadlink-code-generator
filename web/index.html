Protocol Byte Map (18 bytes = 144 bits)

PosNamePurpose
0–1HeaderFixed device ID (C2 3D)
2Fan speedEncodes fan level: auto→BF, quiet→FF, low→9F, medium→5F, high→3F, powerful→3F. Dry mode auto→1F
3CheckComplement of byte 2: FF − B2 (error detection)
4Mode + Temp Upper nibble (bits 7–4) = temperature (16–30°C). Encoding varies by mode:
cool: [4]=b1, [5]=b2, [6]=b3&~b2, [7]=1 if t≥25
heat: [4]=b1, [5]=b2&b3, [6]=b3⊕b2, [7]=1 if t≥25
dry: [4]=b1, [5]=b2&~b1, [6]=b3, [7]=1 if t≥25
fan_only: always E4h (temperature not applicable)
Some adjacent temps share the same code.
Lower nibble (bits 3–0) = mode: 0=cool, 4=fan_only/dry, C=heat
5CheckComplement of byte 4: FF − B4 (error detection)
6–11RepeatCopy of bytes 0–5 (for reliability)
12Footer markerFixed D5
13Fan %Fan speed as percentage (quiet=1, low=40, medium=60, high=80, powerful=100, auto=102)
14–16ReservedUsually 00; set to 10 at 16°C, 02 at fan speed powerful
17ChecksumSum(B12..B16) % 256

How to Use

1. Capture IR signals — Use uv run python main.py with your Broadlink device. Press a button on your AC remote and copy the Base64 output.
2. Analyze raw timings — Paste raw timing data (from captures/*.txt) into the Input tab. Click "Visualize" to see the waveform and "Analyze" to decode the protocol.
3. Build SmartIR JSON — Switch to the SmartIR tab. Fill in manufacturer/model info. Paste each captured Base64 command and name it. Click "Generate" then "Download".
SmartIR JSON files go in custom_components/smartir/codes/<type>/ inside your Home Assistant config directory.
